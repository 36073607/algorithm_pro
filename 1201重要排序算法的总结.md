# **重要排序算法的总结**



1.稳定性

1. 排序算法的稳定性是指：同样大小的样本在排序之后不会改变原始的相对次序
2.  稳定性对基础类型对象来说毫无意义；稳定性对非基础类型对象有意义，可以保留之前的相对次序



2.主要算法时间、空间、稳定性总结：

|     名称      |   时间    |  空间   | 稳定性 |
| :-----------: | :-------: | :-----: | :----: |
| SelectionSort |  O(n^2)   |  O(1)   |   无   |
|  BubbleSort   |  O(n^2)   |  O(1)   |   有   |
| InsertionSort |  O(n^2)   |  O(1)   |   有   |
|   MergeSort   | O(n*logn) |  O(n)   |   有   |
|   QuickSort   | O(n*logn) | O(logn) |   无   |
|   HeapSort    | O(n*logn) |  O(1)   |   无   |
|   CountSort   |   O(n)    |  O(m)   |   有   |
|   RadixSort   |   O(n)    |  O(m)   |   有   |



3.补充

1. 基于比较的排序，时间复杂度O(n*logn)，空间复杂度低于O(n)，还具有稳定性的排序算法目前没有找到

2. 总结

​        1.数据量非常小的情况下可以做到非常迅速：插入排序

​        2.性能优异、实现简单且有利于改进（面对不同业务可以选择不同划分策略）、不在乎稳定性：随机快排

​        3.性能优异、不在乎额外空间占用、具有稳定性：归并排序

​        4.性能优异、额外空间占用要求O(1)、不在乎稳定性：堆排序
